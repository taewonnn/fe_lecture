0902 CSS 특강 Cascading Style Sheet 1. 페이지 제작 시 고려 사항 - 레이아웃 구성 - 웹사이트가 몇 개의 단락으로 구성되어 있는가? - 단락을 기준으로 우선 뼈대만 만들어두기 - head -> footer -> contents 순으로 작성 - 디테일(폰트, 색상 등...)은 최대한 마지막에 작성 2. 용어 - Type Selector(태그 선택자) Ex. dib{ } - Property(속성) Ex. background / width / font ... - Value(값) Ex. orange / 300px / bold ... - Block level elements Ex. div / ul / li / h1 ~ h6 / p - Inline elements Ex. div ul li / h1 ~ h6 / p 내부에 있는 텍스트 / 이미지 - initial value CSS에서는 초기값이 설정되어 있어, width값을 설정 안해도 디폴트값으로 설정되어 있는 상황임 - user agent stylesheet 브라우저가 제공하는 기본 스타일 Ex. body 요소에 적용되어 있는 margin 8px - width : auto; / height : auto; 의미 Ex. div{ background-color:orange; width: auto; height: auto; } - 컨텐츠가 있을 때 div 요소가 넓이를 가지는 이유 width 속성은 initial value(초기값)으로 auto를 갖기 때문에 최대 넓이를 가지는 것 - width : auto; 의미 브라우저가 계산한 margin / padding을 고려하여 유연하게 부모가 제공하는 컨텐츠 영역 넓이만큼 가득채움 * width : 100%의 차이 부모가 제공하는 컨텐츠 영역 최대 넓이만큼 가득차지만 단순히 계산된 픽셀값 따라서 margin / padding값을 지정해주면 부모 요소를 벗어날 수 있음. * height: auto; 의미 자식이 가지는 컨텐츠 높이만큼 유연하게 가짐 width와는 다르게 자식이 기준이 됨 - Box-sizing : content-box(initial value) border-box -> width / height가 패딩 /보더가 포함되게 - margin-left: auto -> 사용가능한 공간을 여백으로 취하는 것 - margin: 0 auto -> 0 : 상하단 / auto : 좌우 - class = attribute(속성) * id와는 달리 중복될 수 있는 점이 특징이고 추가도 가능 Ex. Github Facebook - .class name : 앞에 .은 *이 생략되어 있으며 *은 전체를 의미 - .header img {} -> header 내부에 있는 모든 이미지(자손 모두 포함)하여 적용 - .header > img {} -> header의 직계 자식만 적용 - Type Selector : HTML 태그 선택자 - class Selector
